# v0.2.0 — Bugfixes, Scoring Tiers, Invite Sharing, Results UI, Leaderboard, What If?

**Overall Progress:** `~82%` (9/11 steps done — Steps 1-7, 9-10 complete, Step 8 tests + Step 11 deploy remaining)

## Context
v0.1.2 shipped with several UX gaps and a settings bug. This release fixes the disabled-select form bug, makes results entry discoverable, adds social sharing for invites, implements tiered scoring, enhances the pre-results leaderboard, and adds a "What If?" simulation for admins.

## Critical Decisions
- **Disabled select fix**: Hidden input preserves value — standard HTML fix
- **Results link**: Pool detail action bar, visible to ADMIN + RESULTS_MANAGER
- **Invite sharing**: Compose `InviteShareButtons` around existing `CopyInviteLink` — don't duplicate clipboard logic
- **Scoring tiers**: 4-tier system (180/90/30/15), 0.6x runner-up multiplier. Seed must explicitly write `runnerUpMultiplier: 0.6` in upsert update blocks (schema default only applies to new rows)
- **Pre-results leaderboard**: "Show only my stats" privacy model — current user sees own completion %, others show sealed badges. No podium/envelope complexity — keep scope minimal.
- **What If?**: Client-side only using `calculateLeaderboard()` (pure function). Uses shadcn Sheet component. Swaps leaderboard entries in-place with "SIMULATION" banner. Admin-only.
- **`?created=1` guard**: Admin-only check + `router.replace()` URL cleanup to prevent dialog re-showing on refresh
- **Prisma queries**: Extract raw `prisma.category.findMany()` from leaderboard page into `src/lib/db/categories.ts`
- **Semantic tokens**: Fix `bg-gray-300/15`, `text-gray-300` violations in LeaderboardTable while we're touching it

## What If? Architecture

```
Server (leaderboard page.tsx)
  ├─ getCategoriesWithNominees(ceremonyYearId)  ← new DB helper
  ├─ Fetch predictions (existing)
  ├─ Calculate real leaderboard (existing)
  ├─ Pass { categories, nominees, predictions, isAdmin } to client
  │
Client (WhatIfSimulator.tsx)
  ├─ Admin clicks "What If?" → Sheet opens
  ├─ Random: pick random nominee per category → mockWinnerIds map
  ├─ Manual: admin selects via dropdowns → mockWinnerIds map
  ├─ Build ScoringInput[] with mock winnerIds
  ├─ Call calculateLeaderboard() (pure function)
  └─ Swap leaderboard entries in-place + "SIMULATION" banner
```

No DB writes. No API calls. Pure client-side computation.
